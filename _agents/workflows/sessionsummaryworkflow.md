# Session Summary Workflow

At the end of a work session, run this workflow to capture what was learned and keep the system improving.

## Steps

1. Review what was accomplished this session (tools built, workflows created/updated, errors resolved)
2. Update the relevant `workflows/*.md` files with any new edge cases, rate limits, or better methods discovered
3. Note any new tools added to `tools/` and their purpose
4. If credentials were added, confirm `.env.example` reflects the new keys (without values)
5. Summarize the session: what worked, what didn't, what to pick up next time
